SD‑DEF Procedures Documentation Set

---

1. 🧩 Ingestion Procedures

Asset Inventory Loading
Purpose: Produce a deterministic AssetSet.  
Inputs: CMDB, cloud APIs, endpoint registry  
Outputs: AssetSet  
Procedure:
1. Query all authoritative asset sources.  
2. Normalize identifiers.  
3. Deduplicate by deterministic hash.  
4. Validate required fields.  
5. Emit AssetSet.

---

Control Catalog Loading
Purpose: Produce ControlSet aligned to NIST‑CSF.  
Inputs: Internal control library  
Outputs: ControlSet  
Procedure:  
1. Load control definitions.  
2. Map each control to NIST‑CSF function/category/subcategory.  
3. Validate uniqueness.  
4. Emit ControlSet.

---

Policy Baseline Loading
Purpose: Produce PolicyBaselineSet.  
Inputs: SOPs, configuration baselines  
Outputs: PolicyBaselineSet  
Procedure:  
1. Parse baseline definitions.  
2. Convert to normalized rule objects.  
3. Validate rule syntax.  
4. Emit PolicyBaselineSet.

---

Threat Model Loading
Purpose: Produce ThreatModel.  
Inputs: Threat intel, attack patterns  
Outputs: ThreatModel  
Procedure:  
1. Load threat classes.  
2. Normalize identifiers.  
3. Map threats to assets and controls.  
4. Emit ThreatModel.

---

3. 🔄 Signal Normalization & Classification Procedures

Signal Normalization
Inputs: All signal streams  
Outputs: NormalizedSignalStream  
Procedure:  
1. Convert timestamps to canonical format.  
2. Normalize severity.  
3. Normalize asset/control references.  
4. Emit normalized signals.

---

Signal Classification
Inputs: Normalized signals  
Outputs: Typed signals  
Procedure:  
1. Apply rule‑based classifier.  
2. Assign type: misconfig, drift, visibility loss, etc.  
3. Emit typed signals.

Inputs: Typed signals, AssetSet, ControlSet
